`GET /api/virtual-classes/v2.0/content-providers/{provider}/programs`

This operation search Programs. It returns only brief information about Programs.

|===

=== link:#resources-fetch-program-video-briefs-controller-i-t-should-fetch-program-videos[Fetch Program Videos]

`GET /api/virtual-classes/v2.0/content-providers/{provider}/programs/{programId}/videos`

This operation returns brief information about Program Videos.

[[_request_headers_5]]
==== link:#_request_headers_5[Request headers]

[width="100%",cols="50%,50%",options="header",]
|===
|Name |Description
|`Accept` |application/json
|`authorization` |Api key for authentication (f.e. 'authorization: apiKey 0a47c3bf-4740-465d-a22e-0b25ef86ddd0')
|===

[[_path_parameters_5]]
==== link:#_path_parameters_5[Path parameters]

[width="100%",cols="25%,25%,25%,25%",options="header",]
|===
|Parameter |Type |Optional |Description
|provider |String |false |Unique ID of the Content Provider.
|programId |String |false |Unique ID of the Program.
|===

[[_query_parameters_5]]
==== link:#_query_parameters_5[Query parameters]

[width="100%",cols="25%,25%,25%,25%",options="header",]
|===
|Parameter |Type |Optional |Description
|exerciserUuid |String |true |Exerciser Unique ID.
|sort |String |true a|
Sort order

Can be one of [`createdAsc`, `createdDesc`, `titleAsc`, `titleDesc`, `instructorAsc` (`instructor`), `instructorDesc`, `locationAsc` (`location`), `locationDesc`].

Category-specific user-set order if not specified or not matching the listed options.
|location (deprecated) |String |true |Location filter. Location names joined by comma (deprecated)
|level (deprecated) |String |true |Workout level filter. Level names joined by comma (deprecated)
|instructor (deprecated) |String |true |Instructor filter. Instructor names joined by comma (deprecated)
|equipment (deprecated) |String |true |Equipment filter. Equipment names joined by comma (deprecated)
|location[] |Array[String] |true |Location filter. Location names.
|level[] |Array[String] |true |Workout level filter. Level names.
|instructor[] |Array[String] |true |Instructor filter. Instructor names.
|equipment[] |Array[String] |true |Equipment filter. Equipment names.
|equipmentReq |String |true a|
Equipment required filter.

Can be one of [`yes`, `no`].

`yes` matches videos with equipment set but not equal 'N/A'.
`no` matches videos without equipment set or set to 'N/A'.
|===

.Deprecation note
[NOTE]
====
`location`, `level`, `instructor`, `equipment` filter query parameters support
2 ways of passing values: as collections (when [] suffix is used) or as
comma-separated strings of values.

Apparently, the comma-separated values don't support commas in filter
values, therefore this way is deprecated in 2.0 and will be removed in future
versions. Do not use it.

Presently both ways can be used. Avoid mixing them as you might get the
unexpected results.
====
